Getting Started

LLM-based Chatbots

  • There are now a large number of LLM-based chatbots to choose from.
  • You get a limited number of messages for free.
  • Heavier use requires a paid subscription.
  • For simple tasks the differences between the top chatbots are minor.
  • Things change very quickly in this area.

What Can LLMs Do?

Capabilities

All of these models combine:

  • Language Skills
  • Knowledge
  • Reasoning

Language Skills

  • Requests and responses are in natural language, no programming language, commands or special syntax needed.
  • Can extract information from unstructured text.
  • Can summarise texts.
  • Can translate between languages.

Experiment. Extract information from a Japanese text.

Knowledge

  • The models have memorised large parts of the internet, including Wikipedia.
  • They can answer questions about many topics.
  • If the model does not know an answer, it may:
    • say it does not know,
    • do a quick web search to find the answer, or
    • make up an answer (“hallucinate”).
  • Always verify important facts. Even confident-sounding answers can be wrong.

Experiments

  • “Please list the three biggest cities in Cabinda.”
  • “What is the capital of Astrogovia?”
  • “What is a Riemannian manifold called where only certain tangent directions are allowed in every point?”

Reasoning

  • Can handle multi-step problems: logic puzzles, debugging code, analyzing arguments, following mathematical derivations.
  • This has much improved in recent models.
  • The models make mistakes! Sometimes they can correct the mistakes when you point them out.

Experiment. “You have coins worth 3, 6, and 7 cents. What is the largest amount you can’t make exactly?”

Strange fact: LLMs can perform extremely impressive tasks while simultaneously struggling with surprisingly simple problems. Often it is difficult to predict which request will fall into which category. 1

Practical Advice

Practical Advice

  • Mental model:
    talking to a PhD student via WhatsApp.
  • Use separate chats for separate topics.
  • Iterate! Ask the model to improve its answer.
  • Attach required information rather than relying on the model’s knowledge. Text-based formats work best, e.g. attach TeX files instead of PDF documents.
  • Begin with general questions to establish context before getting specific.

Practical Advice (continued)

  • LLMs are not good with numbers!
    Ask for a python script to do calculations if needed.
  • If you need a figure, ask for a SVG file (text-based format), a Python script that generates the figure, or a LaTeX TikZ code snippet.

Ideas for Prompts

  • “Please review … . If you could make a single change, what would it be?”
  • “Please turn this into a LaTeX table.”
  • “… Suggest three options.”

Programming

Chatbots Can Code

  • The models know how to use MatPlotlib, NumPy, Pandas, etc.
  • Important: iterate!
    Paste any error messages back into the chat to get them fixed.

Example. “Let X be the solution of the SDE dXt = −1/2sin (Xt)dt + dBt with X0 = 0. Please give me a Python script which computes a Monte Carlo estimate for P(X_{10} > 1). Use the Euler-Marujama scheme to simulate the SDE. Think about what Monte Carlo sample size you need.”

Example. “Please write a Python script which uses matplotlib to plot the coastline of Greenland. Overlay the plot with a 1 degree longitude/latitude grid. Make Greenland white and the sea blue. Mark the location at 66°01′40″N 53°32′00″W using a red circle.”

Maths

Chatbots Can Do Some Maths

  • Simple things work.
  • Longer proofs often fail, because the models make mistakes too often and don’t get all the steps right at the same time.

Experiment. “I am studying Galois theory. If f and g are separable polynomials over K, then is the composition f(g(x)) also separable?” (Thanks, Kevin Houston!)

Experiment. “The value C(2n, n)/22n is the probability of getting n heads in 2n coin tosses. Express the asymptotic behavior of this quantity once using Stirling’s formula from the preceding problem and once using Wallis’s product, and determine from this the value of the constant in Stirling’s formula.”

Experiment. “Let ϕ: ℝ² → ℝ be a continuous function such that there exists a mapping f : ℝ → ℝ with ϕ(x₁, x₂) = f(x₁) · f(x₂) for all x₁, x₂ ∈ ℝ, and there exists a function g: ℝ⁺ → ℝ with ϕ(x) = g(|x|) for all x ∈ ℝ², and ϕ(x) > 0 for all x ∈ ℝ². Show that this is satisfied if and only if there exist numbers a > 0, b ∈ ℝ such that ϕ(x) = a · exp(b · |x|²).”

A research chatbot has just achieved gold medal standard at the IMO.

LLMs vs. Specialized Mathematical Software

  • Use LLMs for: explaining concepts, generating LaTeX, writing computational scripts, exploring ideas
  • Use specialized tools for: symbolic computation requiring precision, formal proof verification, large-scale numerical computations

Use Cases

What Works?

I list some use cases where I personally found LLMs to be very useful.

Working with text:

  • Polishing writing, for example “Please shorten this text down to two paragraphs.”
  • “Please review this … Can you spot any typos or mistakes?”
  • “Please merge these two documents into one. Remove any duplicate information, and make sure the result is coherent.”

Example. 1

I am writing a presentation about LLM-based chatbots for my colleagues in the maths department at a UK university. The title is "Using AI Effectively".

Please review the attached slides.  Can you find any typos or mistakes?  If you could add one thing, what would it be?  If you could remove one thing, what would it be?

Note: The transcript 1 features a hallucination near the end.

“On a Mac laptop, how can I prevent the display from going to sleep during a presentation?” 1

Working with code:

  • Write/update a Python script for matplotlib figures: “Please move the red line slightly to the right and make it dashed.”
  • Paste an error message into the chat (without any scaffolding), attach the code. Most of the time you will get an instant solution. (Also works for LaTeX errors.)

LLMs are good (and very patient) at explaining things.

  • Learning some new software. For example, I didn’t know about “reveal.js” before this talk.
  • Download a paper (TeX source) from arXiv and have Claude explain it to me.

Finding references:

  • “I want to learn about reproducing kernel Hilbert spaces. Where do I start? Please select some good references for me.”

Consolidating information from multiple web pages:

  • “Please do a web search: what is the current state of the drought in northern England?”

What Doesn’t Work?

LLMs can handle simple calculations, but they struggle with arithmetic and complex calculations. They can do some algebra, but often make mistakes.

  • Instead of asking “Please invert this matrix”, ask for a Python script that does the work.

LLMs make mistakes, so they are not good for anything which requires absolute precision.

Coding works well, because compiler/runtime errors catch many mistakes and the models can fix them. Maths does not have this automatic feedback loop.

LLMs are not good at telling apart the context of different pieces of text in their input.

  • If one attached document has actual policies, and another one has proposed new policies, the model may confuse the two when answering questions.
  • Since often both instructions and data are English language text, the model may not be able to tell apart the two. Users can exploit this by disguising harmful instructions as normal content, tricking the chatbot. 1 2 3
